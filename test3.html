Что такое Alpha.SDK?

Инструмент для интеграции с Альфа платформой, обеспечивающий доступ к оперативным и историческим данным, а также событиям. Основой SDK является асинхронная модель, позволяющая выполнять длительные операции без блокировки основного потока приложения. Это делает SDK идеальным инструментом для создания отзывчивых пользовательских интерфейсов и эффективных сервисов.

Ключевые концепции

Для эффективного использования SDK важно понимать несколько ключевых концепций, лежащих в его основе.

Асинхронная модель

Большинство операций в SDK, требующих взаимодействия с сервером, могут быть выполнены асинхронно. Методы с суффиксом _async (например, connect_async, get_value_async) немедленно возвращают объект асинхронной операции (например, EmptyOperation, VqtOperation). Это позволяет приложению продолжать работу, не дожидаясь завершения операции.

Статус операции можно отслеживать с помощью метода get_status(), который возвращает значение из перечисления AsyncOperationStatus (SCHEDULED, SUCCESS, FAILED, CANCELLED). Результат можно получить, вызвав метод get_result(), который заблокирует выполнение до завершения операции, или использовать неблокирующий вызов try_get_result().

Каналы данных (DataPipe)

DataPipe — это унифицированный механизм для получения потоковых данных от сервера. Когда вы запрашиваете набор данных (например, список свойств элемента или результат подписки на изменения), SDK возвращает объект-потомок DataPipe.

Данные из канала считываются последовательно с помощью метода fetch_next(timeout). Канал может быть двух типов:

  • Результат запроса: Канал содержит конечное число элементов. После считывания всех данных метод is_over() вернет True.
  • Подписка: Канал постоянно получает новые данные (например, изменения значений тега). Метод is_over() не вернет True, пока подписка активна.

Работа с событиями

SDK предоставляет инструменты для работы с системными событиями Alpha.Domain. Вы можете как получать события в реальном времени, так и запрашивать их из истории.

  • Оперативные события: Метод RuntimeNode.subscribe_events() позволяет подписаться на поток событий, происходящих на сервере.
  • Исторические события: Метод HistoryNode.create_event_read_session() создает сессию для чтения событий из архива за указанный период.

Для обоих случаев можно использовать объекты EventFilterByName или EventFilterById, чтобы указать, от каких именно источников (тегов) или ветвей конфигурации вы хотите получать события.

Установка

Шаг 1: Системные требования

  • Операционная система: Windows 10 или новее (x64); GNU/Linux.
  • Python: Python 3.6+ (x64).

Шаг 2: Установка Alpha.Domain.Client

1
Перейдите на портал технической поддержки (требуется регистрация).
Портал поддержки Automiq
2
Найдите раздел ComponentSet > Alpha.Domain.Client и скачайте дистрибутив Alpha.Domain.Client-1.1.2+b1.g7c096c73 (x64).
3
Запустите установочный файл от имени администратора и следуйте инструкциям мастера установки.
4
По умолчанию компонент устанавливается в: 
C:\Program Files\Automiq\Alpha.SDK\Alpha.Domain.Client\

Шаг 3: Установка Python-пакета

1
Скачайте Python 3.6 или выше. При установке обязательно отметьте опцию "Add Python to PATH".
Официальный сайт Python
2
В командной строке перейдите в папку с установочным пакетом: 
cd "C:\Program Files\Automiq\Alpha.SDK\Alpha.Domain.Client\package"
3
Установите whl-пакет: 
python -m pip install alpha_domain_pyclient-1.1.2+b1.g7c096c73-cp36-abi3-win_amd64.whl

Получение и инициализация элементов

Пользователь вводит имена тегов, скрипт получает объекты и инициализирует их.

requested_items = runtime_node.get_items(item_names)
items = []
for item in requested_items:
    item.init()
    if item.status_info.item_status == ng.ItemStatus.INITIALIZED:
        items.append(item)
Ключевые моменты:
  • get_items(names): получает объекты Item по именам.
  • item.init(): инициализирует элемент - обязательный шаг.
  • ItemStatus.INITIALIZED: статус готовности элемента.

Чтение текущих значений

Итерация по элементам и запрос их текущего значения (VQT).

for item in items:
    try:
        vqt = item.get_value()
        print(f"- {item.name}: {vqt.value}")
        data_logger.write_data(item.name, vqt.value, vqt.quality)
    except Exception as e:
        print(f"X Ошибка: {e}")

Важные рекомендации

Безопасность и надежность:

  • Тестирование: всегда тестируйте скрипты в изолированной среде.
  • Обработка ошибок: используйте try-catch блоки для всех операций SDK.
  • Управление соединениями: правильно закрывайте подключения в finally блоках.
  • Логирование: ведите подробные логи для диагностики проблем.

Рекомендуемые инструменты разработки:

  • IDE: PyCharm Professional, VS Code с Python Extension.
  • Отладка: используйте встроенные отладчики для пошагового анализа.
  • Версионирование: Git для управления версиями ваших скриптов.
  • Виртуальные среды: venv для изоляции зависимостей проектов.
